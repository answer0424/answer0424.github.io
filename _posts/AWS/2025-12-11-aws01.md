##  무엇이 AWS인가

- AWS는 컴퓨팅, 스토리지, 데이터베이스, 분석, 네트워킹, 모바일, 개발자 도구, 관리 도구, IoT, 보안, 엔터프라이즈 어플리케이션 등을 포함한 방대한 글로벌 클라우드 서비스 세트를 제공
- 전체적으로 200개 이상의 AWS 서비스를 온디맨드 방식으로 제공하며, 사용자는 몇 초 만에 서비스 프로비저닝이 가능

<hr>

## AWS를 사용하는 이유 / 클라우드의 이점

- 선결제 고정 비용 없이 필요할 때만 리소스를 사용하고 비용을 지불하는 즉 pay-as-you-go 방식
- 과거처럼 수주 혹은 수개월 전에 서버나 인프라를 계획하고 구입할 필요없이, 몇 분 만에 수백 또는 수천 개 서버를 즉시 가동할 수 있음
- 이를 통해 기업, 스타트업, 중소기업, 공공 부문 등 다양한 고객이 비즈니스 요구 변화에 신속하게 대응 가능

<hr>

## AWS 서비스 구성 개요

- 컴퓨팅(Compute)
- 스토리지(Storage)
- 데이터베이스(Database)
- 네트워킹 & 콘텐츠 전송(Networking & Content Delivery)
- 보안, 자격 증명 및 규정 준수 (Security, Identity, & Compliance)
- 분석(Analytics)
- 머신 러닝/인공 지능(Machine Learning/AI)
- 사물 인터넷(IoT)

#### AWS 서비스에 접근하거나 관리하는 방식

- 웹 UI 기반: AWS Management Console
- 명령줄 도구: AWS Command Line Interface (AWS CLI)
- 프로그래밍 인터페이스(SDKs): 여러 언어나 플랫폼에 맞춘 SDK 제공

<hr>

## 주요 관점 & 설계 철학

-  AWS를 사용하면 고정된 자본 비용 대신 필요한 만큼만 지불하는 가변 비용 구조로 IT 인프라를 운영할 수 있음
-  또한 수요 변화에 맞게 빠르게 리소스를 확장하거나 축소할 수 있어, 인프라에 대한 기회과 조달의 부담이 크게 줄어듦
-  AWS의 글로벌 인프라와 광범위한 서비스 카탈로그 덕분에 , 스타트업부터 대기업, 공공기관에 이르기까지 다양한 규모와 요구의 조직이 유연하게 클라우드로 전환할 수 있음
