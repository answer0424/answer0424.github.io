## AWS 애플리케이션 통합

> AWS 애플리케이션 통합은 마이크로 서비스, 분산 시스템 및 서버리스 애플리케이션 등에서 분리된 구성 요소 간의 통신을 지원하는 서비스 제품군
>
> 이를 통해 전체 아키텍처를 리팩토링하지 않고도 시스템 요소들을 느슨하게 결합하여 변경 영향을 줄이고, 새 기능을 빠르게 릴리스할 수 있도록 함

<hr>

## 애플리케이션 통합 지원 서비스 목록

#### AWS Step Functions

> 완전 관리형 시각적 워크플로 서비스

- 분산 애플리케이션과 마이크로 서비스 구성 요소를 단계별로 조정할 수 있음
- 각 단계 실행 상태를 추적하고 오류 시 재시도/디버깅을 쉽게 할 수 있음

<br>

#### Amazon AppFlow

> SaaS 애플리케이션과 AWS 서비스 간 데이터 통합/전송을 지원하는 완전관리형 서비스

- 데이터 전송 시 암호화, 필터링 및 변환 기능을 제공
- 퍼블릭 인터넷 대신 AWS PrivateLink를 활용해 보안도 강화할 수 있음

<br>

#### AWS B2B 데이터 교환

> 전통적인 EDI를 JSON 또는 XML로 자동 변환하여 다른 데이터 시스템과 통합을 쉽게 함

- 거래 파트너 온보딩, 관리 및 변환 과정을 자동화해 시간/비용 절감을 제공


<br>

#### Amazon EventBridge

> 서버리스 이벤트 버스 서비스

- AWS 서비스, SaaS 애플리케이션, 사내 애플리케이션에서 생성된 이벤트를 라우팅
- 이벤트 기반 아키텍처 구축을 용이하게 함

<br>

#### Amazon Managed Workflows for Apache Airflow (MWAA)

> Apache Airflow의 완전 관리형 로케스트레이션 서비스

- 워크플로 정의, 스케줄링 및 운영을 엔드 투 엔드로 간소화함
- 확장성/가용성/보안이 자동으로 제공

<br>

#### Amazon MQ

> Apache ActiveMQ & RabbitMQ 기반의 메시지 브로커 서비스

- 다양한 프로그래밍 언어 및 표준 메시징 프로토콜을 지원
- 기존 애플리케이션 메시징 코드 변경 없이 클라우드로 마이그레이션할 수 있음

<br>

#### Amazon Simple Notification Service (SNS)

> 완전 관리형 pub/sub 메시징 서비스

- 여러 구독자에게 대규모 메시지를 전송할 수 있음
- 모바일 푸시, SMS, 이메일 등 다양한 알림 채널로 확장 가능

<br>

#### Amazon Simple Workflow Service (SWF)

> 완전 관리형 상태 추적형 작업 오케스트레이션 서비스

- 복수 단계로 구성된 백그라운드 작업을 빌드, 실행, 확장할 수 있음
- 작업 실패 시 복구 및 재시도 관리를 도움

<br>

#### 주요 목적

> AWS 애플리케이션 통합 서비스는 다음 목표를 위해 설계

- 구성 요소 간 느슨한 결합을 지원하여 애플리케이션 변경 영향 최소화
- 분산/서버리스 시스템 간 통신/데이터 전달 간소화
- 이벤트 기반 아키텍처 및 워크플로 오케스트레이션을 쉽게 구축할 수 있도록 지원
