## Amazon EC2란 무엇인가?

> Amazon EC2 (Elastic Compute Cloud)는 AWS 클라우드에서 온디맨드로 확장 가능한 컴퓨팅 용량을 제공하는 서비스
>
> 이를 통해 물리 서버 구입 없이 원하는 수의 가상 서버를 실행하고 보안/네트워크/스토리지 설정을 통합 관리할 수 있음

<br>

#### 특징

- 필요할 때 컴퓨팅 자원을 확장/축소할 수 있음
- 다양한 인스턴스 유형을 선택하여 워크로드에 맞는 리소스 구성 가능
- 보안, 네트워크 설정, 스토리지 관리 등 인프라 유연성 제공

<hr>

## Amazon EC2의 핵심 구성 요소

#### 인스턴스 (Instance)

- EC2 인스턴스에 설치할 운영체제 및 소프트웨어 구성을 포함한 템플릿
- 인스턴스는 이 AMI를 기반으로 생성됨

<br>

#### Amazon Machine Image (AMI)

- EC2 인스턴스에 설치할 운영체제 및 소프트웨어 구성을 포함한 템플릿
- 인스턴스는 이 AMI를 기반으로 생성

<br>

#### 인스턴스 유형 (Instance Types)

- CPU, 메모리, 네트워크, 스토리지 구성의 조합으로 성능 특성이 다름
- 워크로드 특성에 맞춰 적절한 인스턴스 유형을 선택할 수 있음

<br>

#### Amazon EBS 볼륨 (Elastic Block Store)

- EC2 인스턴스에 연결할 수 있는 영구 블록 스토리지
- 인스턴스를 중지해도 데이터는 유지

<br>

#### 인스턴스 스토어 볼륨 (Instance Store)

- 인스턴스와 함께 제공되는 임시 로컬 스토리지임
- 인스턴스 중지/종료 시 데이터는 사라짐

<br>

#### 키 페어 (Key Pair)

- SSH 또는 RDP 접속을 위한 공개/비밀 키 쌍임
- 공개 키는 AWS에서 관리하고, 개인 키는 사용자 측에서 안전하게 보관

<br>

#### 보안 그룹 (Security Group)

- 인스턴스를 보호하기 위한 가상 방화벽
- 포트/프로토콜/소스 IP 범위를 설정하여 인바운드/아웃바운드 트래픽을 제어

<hr>

## EC2와 함께 사용할 수 있는 AWS 서비스

- Amazon EC2 Auto Scaling: 트래픽 변화에 맞춰 EC2 인스턴스 수 자동 조정
- AWS Backup: EC2 및 EBS 볼륨 백업 자동화
- Amazon CloudWatch: 인스턴스/스토리지 성능 / 로그 모니터링
- Elastic Load Balancing: 여러 인스턴스로 트래픽 분산
- Amazon GuardDuty: 위협 탐지 및 보안 모니터링
- EC2 Image Builder: 보안 이미지 생성/관리 자동화
- AWS Launch Wizard: 복잡한 애플리케이션 배포 자동 구성
- AWS Systems Manager: EC2 운영 관리 및 자동화
