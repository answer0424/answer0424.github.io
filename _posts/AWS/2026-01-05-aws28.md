## 개요

- AWS Management Console을 사용해 EC2 인스턴스 시작
- 인스턴스에 접속
- 인스턴스 정리

<hr>

## 핵심 구성 요소

| 구성 요소 | 설명 |
|-|-|
| 이미지 (AMI) | 운영 체제 포함 템플릿, 인스턴스 시작 기반 |
| 키 페어 | SSH/RDP 연결 시 본인 인증에 사용하는 공개/프라이빗 키 쌍 |
| 네트워크 | AWS 계정 전용 가상 네트워크, 기본 VPC가 자동 제공됨 |
| 보안 그룹 | 인스턴스 트래픽 제어를 위한 가상 방화벽 역할 |
| EBS 볼륨 | 루트 디스크 등 영구 저장용 블록 스토리지 |


<hr>

## 1단계 - EC2 인스턴스 시작

1. EC2 콘솔 접속 - [https://console.aws.amazon.com/ec2/](https://console.aws.amazon.com/ec2/)에서 로그인
2. 리전 선택 - 상단 리전 메뉴에서 가까운 리전을 선택
3. 인스턴스 시작 선택 - 인스턴스 시작 버튼 클릭
4. 이름 및 태그 설정 - 인스턴스 이름을 입력하여 식별
5. AMI 선택 - 빠른 시작 / 프리티어 가능 AMI 선택
6. 인스턴스 유형 선택 - 프리티어 범위 내 인스턴스 유형 선택
7. 키 페어 설정 - 기존 키 선택 또는 새 키 페어 생성 / 키 페어 없이는 SSH/RDP 접속 불가능
8. 네트워크/보안 그룹 
   - 기본 VPC / 서브넷 선택
   - 테스트용 보안 그룹은 일시적으로 `0.0.0.0/0` 설정 가능
   - 운영 환경에서는 개별 IP만 허용하도록 제한 필요
9. 스토리지 확인 - 루트 EBS 볼륨은 기본 구성으로 설정됨
10. 확인 후 시작 - 요약 화면에서 설정 확인 ➡️ 인스턴스 시작 클릭
11. 상태 확인 - 인스턴스가 `pending`에서 `running` 상태로 변경되면 준비 완료

<hr>

## 2단계 - 인스턴스에 연결

#### SSH 접속

1. 콘솔에서 인스턴스 선택 ➡️ Connect 클릭
2. SSH 클라이언트 탭 선택
3. 다운로드 받은 `.pem` 키 파일 권한 설정 (chmod)
4. SSH 명령 복사 후 실행

```bash
ssh -i key-pair-name.pem ec2-user@public-dns-name
```

- OS에 따라 사용자가 다를 수 있음
