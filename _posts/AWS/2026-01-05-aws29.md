## 보안 (Security)

#### IAM 역할 및 페더레이션

- 가능한 경우 IAM 역할 및 외부 ID 제공 업체와 페더레이션을 이용해 AWS 자원 및 API 액세스를 관리함

> 권한을 최소화하고 역할 기반 접근을 사용하여 보안성을 높임

<br>

#### 보안 그룹 최소 허용 규칙

- 인바운드/아웃바운드 규칙은 최소한으로만 열어두고 필요한 IP/포트만 허용

> 광범위한 접근(예: `0.0.0.0/0`)은 보안 위험을 초래할 수 있음

<br>

#### 운영 체제 및 소프트웨어 패치

- 인스턴스 내부의 OS와 애플리케이션을 정기적으로 업데이트 및 패치

> 취약점 위험을 줄이고 최신 보안 업데이트를 적용해야 함

<br>

#### 보안 검사 자동화

- Amazon Inspector를 사용해 취약성/구성 오류/네트워크 노출을 자동 스캔함
- AWS Security Hub CSPM 제어를 활성화해 중앙에서 보안 규정 준수 상태를 모니터링 함

<hr>

## 스토리지 (Storage)

#### 루트 디바이스 유형 이해

- 루트 볼륨 유형은 데이터 유지/백업/복구 전략에 영향을 줌

<br>

#### EBS 볼륨 분리

- 운영 시스템과 데이터용 EBS 볼륨은 분리

> 인스턴스를 종료하더라도 데이터가 유지되게 구성해야 함

<br>

#### 인스턴스 스토어 사용 주의

- 임시 데이터용으로만 사용하고, 인스턴스 중단/종료 시 데이터가 사라진다는 특성을 이해

<br>

#### EBS 암호화

- EBS 볼륨과 스냅샷을 암호화해 데이터 보안을 강화

<hr>

## 리소스 관리 (Resource Management)

#### 태그 (Tagging)

- 인스턴스 메타데이터 및 사용자 정의 태그를 활용해 환경을 추적하고 비용/운영 단위로 구분

<br>

#### 서비스 할당향 (Quotas)

- EC2 리소스 제한을 확인하고, 실제 필요 시 미리 증가 요청을 함

<br>

#### AWS Trusted Advisor

- Trusted Advisor를 통해 비용 최적화, 성능 개선, 보안 권장사항을 확인하고 적용
