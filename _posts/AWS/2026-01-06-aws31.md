## 개요

> EC2 인스턴스를 생성할 때 가장 먼저 선택하는 항목이 AMI (Amazon Machine Image)
>
> AMI는 인스턴스의 운영체제와 기본 소프트웨어 구성을 포함하는 템플릿

<hr>

## AMI란?

- AMI는 EC2 인스턴스를 시작할 때 사용하는 이미지 템플릿
- 운영 체제, 애플리케이션 서버, 런타임 등이 미리 설치된 형태로 제공되며, 이를 기반으로 여러 인스턴스를 생성할 수 있음

<hr>

## AMI 찾기 - AWS 콘솔에서 

#### 빠른 시작 (Quick Start)

> AWS에서 일반적으로 추천하는 기본 AMI 목록

- 예) Amazon Linux, Ubuntu, Windows Server 등 주요 OS 이미지
- 프리 티어 가능 AMI는 Free Tiew 표시가 함께 나타남

<br>

#### AWS Marketplace

> AWS Marketplace에 등록된 커뮤니티/서드파티 제공 AMI를 검색할 수 있음

- 상용 소프트웨어 포함 AMI
- 일부는 유료 요금/라이선스가 포함될 수 있음

<br>

#### 내 AMI (My AMIs)

- 내가 생성한 커스텀 AMI
- 다른 계정에서 공유된 AMI도 이 목록에서 볼 수 있음

<hr>

## AMI 유형 이해하기

| AMI 카테고리 | 설명 |
|-|-|
| AWS Quick Start AMIs | AWS가 공식 제공하는 OS 이미지 |
| AWS Marketplace AMIs | 서드 파티/상용 이미지 선택 가능 |
| My AMIs | 내 계정 또는 공유 받은 이미지 |
| Community AMIs | 공개 커뮤니티 제공 이미지 |


<hr>

## AMI 선택시 유의사항

#### 신뢰된 이미지 선택

- Quick Start 또는 AWS Marketplace의 Verified Publisher 이미지가 안전성/업데이트 측면에서 권장됨

<br>

#### 요금/라이선스 확인

- Marketplace AMI의 경우 운영체제 + 소프트웨어 라이선스 비용이 포함될 수 있음
