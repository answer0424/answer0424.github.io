## 기본 개념

- AMI: EC2 인스턴스를 시작할 때 사용하는 템플릿 이미지
- EBS-backed AMI: 루트 디바이스가 EBS 볼륨인 AMI. 해당 유형이 현재 대부분 사용됨

<hr>

## 생성방식 

#### 1. 인스턴스에서 AMI 생성

> 이미 실행 중인 EC2 인스턴스에서 직접 AMI를 만듦

<br>

#### 2. 스냅샷에서 AMI 생성

> 루트 볼륨의 EBS 스냅샷이 있다면, 이를 기반으로 AMI를 생성

<hr>

## 프로세스 개요 - 인스턴스에서 생성

#### 1. 기존 AMI 선택

- Marketplace, VM Import/Export, 기타 이미 접근 가능한 AMI를 선택해 인스턴스를 시작

#### 2. 인스턴스 사용자 지정

- 설치할 소프트웨어, 데이터 복사, 불필요 파일 삭제 등 커스터마이징을 수행

#### 3. AMI 생성

- 인스턴스를 일시 중지하고 파일 시스템의 일관성을 유지한 후 AMI 생성이 진행됨
- 옵션에 따라 재부팅 없이 생성할 수도 있음

#### 4. EBS 스냅샷 생성

- 루트 볼륨 및 연결된 추가 EBS 볼륨의 스냅샷이 자동 생성됨
- 이 스냅샷은 이후 AMI에서 인스턴스를 생성할 때 사용됨

#### 5. 새로운 AMI 등록

- 생성 완료 후 자동으로 AMI가 등록

#### 6. 새 AMI로부터 인스턴스 시작

- 생성된 AMI를 이용해 신규 EC2 인스턴스를 실행할 수 있음
- 이때 AMI의 스냅샷으로 EBS 루트 볼륨이 새로 생성됨
