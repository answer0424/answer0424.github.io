## 인스턴스 유형 선택이 중요한 이유

> 인스턴스 유형은 EC2 컴퓨팅 성능(CPU), 메모리, 스토리지, 네트워크 성능을 결정
>
> 적절한 유형을 선택하면 비용 효율성과 성능 최적화를 동시에 달성할 수 있음

---

## 인스턴스 유형 찾기: 주요 방법


#### 콘솔에서 찾기(Instance Types)

- 절차 요약

1. Amazon EC2 콘솔에 로그인
2. 상단 리전 선택에서 워크로드를 배치할
3. 왼쪽 탐색에서 인스턴스 유형 메뉴로 이동함
4. 표시할 속성을 선택
5. 필터를 통해 요구 조건에 맞는 리스트로 축소함
6. 필요하면 여러 유형을 체크하여 비교 뷰로 비교
7. 최종 선택된 유형으로 인스턴스를 생성할 수 있음

<br>

#### AWS CLI로 설명(describe-instance-types)

```bash
aws ec2 describe-instance-types \
    --instance-types t3.micro \
    --region ap-northeast-2
```

> 이 명령은 지정한 인스턴스 유형의 전체 세부 정보를 반환함
>
> 필요한 속성을 쿼리 옵션으로 선택하여 출력할 수도 있음

#### AWS CLI로 찾기

- 가용 영역 기준
- 메모리 크기 기준
- 인스턴스 스토리지 유무 기준
- 최대 절전 모드 지원 여부

## 정리

| 방법 | 쓰임 |
|---|---|
| AWS Console | 시각적 탐색, 속성 기반 필터링 |
| AWS CLI | 스크립트/자동화용 탐색 및 상세 조회 |
| PowerShell | Windows 환경에서 자동화 검색 |
| 파인더/Optimizer | 추천 기반 유형 제안 |
