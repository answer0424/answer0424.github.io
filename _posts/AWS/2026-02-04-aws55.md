## 권장 사항이란?

> Compute Optimizer는 CloudWatch 지표와 인스턴스 사양을 분석하여, 현재 워크로드에 대해 다음과 같은 인스턴스 상태를 평가하고 권장 타입을 제공


| 분류 | 의미 |
|---|---|
| Under-provisioned(과소 프로비저닝) | 워크로드의 성능 요구를 충족하지 못함 |
| Over-provisioned(과대 프로비저닝) | 성능 요구는 충족되지만 불필요한 자원이 존재함 |
| Optimized(최적화됨) | 현재 인스턴스가 적합하게 프로비저닝 되어 있음 |
| None(없음) | 권장 사항 생성 불가 |


---

## 권장 사항 출력 형식

- 현재 인스턴스 유형
- 결과 분류
- 권장 인스턴스 옵션
- 각 권장 옵션의 사양, 예상 비용, 성능 리스크
- CloudWatch 기반 사용률 그래프 비교

---

## 권장 사항 적용 절차

#### Compute Optimizer 활성화

> 먼저 Compute Optimizer를 옵트인해야 권장 사항을 생성할 수 있음

<br>

#### EC2 콘솔에서 확인

- EC2 인스턴스 목록에서 특정 인스턴스를 선택
- 세부 정보 영역에서 Compute Optimizer 조사 결과 링크 선택
- 권장 사항 페이지에서 결과 확인

<br>

#### 결과 평가 후 행동 결정

> 권장 사항은 과거 14일간 메트릭을 기반으로 생성되므로, 단기 과부하/스파이크 상황이 있다면 실제 워크로드 특성을 추가로 확인해 결정해야 함

---

## 권장 사항 평가 시 고려할 점


- 미래 사용량 예측은 반영되지 않음: 권장 사항은 과거 매트릭 기반 분석이므로 
- 지표 데이터 충분성: Compute Optimizer는 CloudWatch 메트릭 최소 요구량을 충족해야 권장을 생성함
- 버스트 가능한 인스턴스 조사: 버스트형 인스턴스의 경우, 버스트 패턴이 지속되는지 여부를 고려하여 권장을 검통해야 함

---

## 요약

- Compute Optimizer는 EC2 인스턴스를 분석해 사용량 패턴 기반 권장 사항을 생성함
- 권장 사항은 과소/과다/최적화 여부로 분류되며, 최대 3개의 권장 옵션을 제시함
- 권장 사항 적용 전에는 향후 사용 패턴과 현재 데이터 기준으로 다시 검토해야 함
