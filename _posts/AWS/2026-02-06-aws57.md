## 변경 가능한 조건

- 인스턴스 루트 볼륨이 EBS 기반 EBS 볼륨이어야 함
  - 인스턴스 저장소 기반 인스턴스는 이 방법으로 유형 변경 불가
- 인스턴스가 `stopped` 상태여야 함
  - 실행 중인 상태에서는 변경할 수 없음
- 변경 대상 인스턴스 유형이 현재 구성과 호환되어야 함
  - 네트워크 드라이버, 가상화 유형, EBS/ENA/NVMe 요구사항 등 호환성 조건이 만족되어야 함

---

## 인스턴스 유형 변경 절차

#### 1. 필요 드라이버 설치

- 새 인스턴스 유형이 ENA 또는 NVMe를 요구하면 인스턴스에 드라이버 설치가 필요할 수 있음

<br>

#### 2. 인스턴스 중지(Stop)

- EC2 콘솔 ➡️ 인스턴스 선택 ➡️ Instance state ➡️ Stop instance
- 인스턴스가 완전히 정지될 때까지 대기

<br>

#### 3. 인스턴스 유형 변경

- 정지 상태에서 Actions ➡️ Instance settings ➡️ Change instance type 선택
- 변경할 새로운 인스턴스 유형 선택
- (선택) 해당 유형이 EBS 최적화를 지원하면 최적화 설정 선택/해제
- (선택) vCPU 옵션 구성
- Change 클릭하여 승인

<br>

#### 4. 인스턴스 시작(Start)

- 유형 변경이 완료되면 Instance state ➡️ Start instance
- 인스턴스가 재시작되며 새 유형으로 실행

---

## 주의사항

#### 인스턴스 IP 주소

- 인스턴스를 중지/시작하면 퍼블릭 IPv4 주소가 변경될 수 있음

<br>

#### 호환성 확인

- 지원되지 않는 인스턴스 유형은 목록에 나타나지 않음 ➡️ 새 인스턴스 시작 후 마이그레이션 필요
