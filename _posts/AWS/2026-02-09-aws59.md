## 핵심 요약

- 버스트 가능한 성능 인스턴스는 기준 CPU 성능을 제공하면서 필요 시 CPU를 일시적으로 더 사용할 수 있는 구조를 갖음
- 이 구조는 CPU 크레딧이라는 메커니즘으로 관리됨
- CPU 사용률에 따라 크레딧 획득/소비가 발생하고 이를 기반으로 버스트 동작이 조절됨

---

## 기본 개념 및 정의

#### CPU 사용률

- 인스턴스에서 사용되는 할단된 CPU 사이클의 비율
- CloudWatch에서 인스턴스 단위로 측정됨

<br>

#### CPU 크레딧

- vCPU 시간 단위로 계산되는 실행 시간 단위
- 예:
  - 1 CPU 크레딧 = vCPU 1개 x 100% x 1분
  - vCPU 2개 x 50% x 2분 1 크레딧에 해당

<br>

#### 기준 사용률

- 크레딧 적립 속도와 소비 속도가 같아지는 CPU 비율
- 베이스라인 이하로 사용하면 크레딧이 적립됨

<br>

#### 획득 크레딧

- 인스턴스가 기준 이상 사용 시 소비되는 크레딧
- 분당 소비량 = vCPU 수 x CPU 사용률 x 1분

<br>

#### 크레딧 누적 한도

- 인스턴스 크기에 따라 최대 적립 가능한 CPU 크레딧 수
- 예: t3.nano 24시간 x 시간당 6크레딧 = 144 크레딧

---

## 버스트 동작 메커니즘

#### 표준 모드

- 크레딧이 있으면 베이스라인 이상으로 CPU 사용 가능
- 크레딧이 없으면 CPU 사용률이 점진적으로 베이스라인으로 떨어짐
- 주로 저평균 CPU 사용량 워크로드에 적합

<br>

#### 무제한 모드

- 크레딧이 부족해도 잉여 크레딧을 사용하여 CPU를 버스트 가능
- 이후 낮은 사용률에서 잉여 크레딧을 청산함
- 24시간 평균 CPU 사용률이 기준을 초과하면 추가 요금 발생

---

## 크레딧 흐름 요약

#### 기준 이하 사용

- CPU 사용률이 기준보다 낮으면 
  - 적립 크레딧 > 소비 크레딧
  - 획득 크레딧 증가
