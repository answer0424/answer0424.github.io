## 핵심 요약

- 버스트 가능한 성능 인스턴스는 기준 CPU 성능을 제공하면서 필요 시 CPU를 일시적으로 더 사용할 수 있는 구조를 갖음
- 이 구조는 CPU 크레딧이라는 메커니즘으로 관리됨
- CPU 사용률에 따라 크레딧 획득/소비가 발생하고 이를 기반으로 버스트 동작이 조절됨

---

## 기본 개념 및 정의

#### CPU 사용률

- 인스턴스에서 사용되는 할단된 CPU 사이클의 비율
- CloudWatch에서 인스턴스 단위로 측정됨

<br>

#### CPU 크레딧

- vCPU 시간 단위로 계산되는 실행 시간 단위
- 예:
  - 1 CPU 크레딧 = vCPU 1개 x 100% x 1분
  - vCPU 2개 x 50% x 2분 1 크레딧에 해당

<br>

#### 기준 사용률

- 크레딧 적립 속도와 소비 속도가 같아지는 CPU 비율
- 베이스라인 이하로 사용하면 크레딧이 적립됨

<br>

#### 획득 크레딧

- 인스턴스가 기준 이상 사용 시 소비되는 크레딧
- 분당 소비량 = vCPU 수 x CPU 사용률 x 1분
