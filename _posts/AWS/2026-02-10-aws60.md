## 핵심 요약

- Unlimited Mode는 버스트 가능 인스턴스에서 CPU 크레딧 부족 이후에도 베이스라인 이상 성능을 유지할 수 있도록 허용하는 옵션임
- 24시간 평균 CPU 사용률이 기준 이상일 경우 추가 요금이 부과될 수 있음
- 실행/중지 상태 모두에서 언제든지 활성화/비활성화 가능하며, 계정/리전/인스턴스 패밀리 단위로 기본값 설정도 가능함

---

## Unlimited Mode 개념

#### 정의

- Unlimited Mode는 버스트 가능 인스턴스에 대한 CPU 크레딧 구성 옵션
- Standard와 무제한 성능 확장을 가능하게 하는 옵션으로 선택할 수 있음
- 실행 또는 중지 상태의 인스턴스에 대해 언제든지 설정 변경 가능

<br>

#### 기본 설정

- AWS 계정 ➡️ 특정 리전 ➡️ 특정 버스트 인스턴스 패밀리 단위로 기본값을 Unlimited로 지정할 수 있음
- 새로운 인스턴스는 해당 기본값을 따르게 됨

---

## 작동 원리

#### 크레딧 소비/잉여 크레딧

- 인스턴스가 베이스라인 이상 CPU를 사용하면 획득 크레딧 ➡️ 소비 크레딧보다 먼저 사용함
- 적립 크레딧이 0이 되면 잉여 크레딧으로 계속 버스트 가능
- 이후 CPU 사용률이 낮아지면 획득한 크레딧으로 잉여 크레딧을 청산

#### 24시간 평균 사용률 개념

- EC2는 롤링 24시간 평균 CPU 사용률을 기준으로 크레딧 사용 및 비용을 계산
- 이 평균 수치가 기준 이상이면 추가 요금이 발생할 수 있음
