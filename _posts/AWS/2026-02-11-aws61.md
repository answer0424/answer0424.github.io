## 핵심 요약

- Standard Mode는 버스트 가능한 EC2 인스턴스의 기본 크레딧 운영 모드임
- 인스턴스는 베이스라인 이하 CPU 사용 시 크레딧을 획득, 초과 사용 시 적립한 크레딧만큼만 버스트 가능
- 적립된 크레딧이 소진되면 CPU는 베이스라인 수준으로 제한됨

---

## Standard Mode 기본 개념

#### 버스트 가능한 인스턴스

> Standard Mode는 T2, T3, T3a, T4g 등 버스트 가능한 인스턴스에 적용됨

#### CPU 크레딧과 베이스 라인

- CPU Credit Balance: 인스턴스가 보유한 크레딧 수
- Baseline Performance: 인스턴스 유형별로 지정된 기준 CPU 비율
- 인스턴스는 베이스라인 이하 사용 시 크레딧 적립, 초과 시 적립된 크레딧 소비 ➡️ 버스트가 가능

--- 

## 크레딧 적립/소비 메커니즘

#### 크레딧 적립

- 인스턴스가 베이스라인 CPU 사용률 이하일 때 적립 발생
- 적립량 = Baseline × vCPU 개수 × 시간
  - 예: t3.small (베이스라인 20%) × 1 vCPU = 12 크레딧/시간

<br>

#### 크레딧 소비

- 인스턴스가 베이스라인 이상 CPU 사용 시 적립된 크레딧을 소비
- 크레딧이 충분하면 베이스라인 이상 성능 유지
- 소진 후에는 베이스라인 이하로 CPU 사용 제한

---

## Standard Mode 작동 흐름

#### 베이스라인 이하 CPU 사용

- 적립 속도 > 소비 속도
- 결과: CPU Credit Balance 증가
