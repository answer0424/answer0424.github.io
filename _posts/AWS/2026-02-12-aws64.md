## 테스트 인스턴스 시작 절차

#### AWS EC2 콘솔 열기

1. AWS Management Console 로그인
2. 서비스 검색에서 EC2 선택
3. EC2 대시보드 진입

<br>

#### EC2 인스턴스 시작 마법사 실행

1. 인스턴스 시작 선택
2. 인스턴스 시작 웹 기반 양식 열림

<br>

#### 인스턴스 이름 및 태깅

- Name 태그 입력
- 태그는 인스턴스 목록에서 필터링/식별에 도움

<br>

#### AMI 선택

- 기본적으로 Amazon Linux 2023 AMI가 선택됨
- 프리티어 사용 가능 표시가 있는 AMI 선택 권장

<br>

#### 인스턴스 유형 선택

- 기본 인스턴스 유형 유지
- 프리 티어 범위 내 선택으로 비용 관리

<br>

#### 키 페어 설정

- 기존 키 페어 선택 또는 새 키 페어 생성 선택
- 프라이빗 키 파일 다운로드 후 로컬 저장 필수
- 추후 SSH 접속 시 사용

```bash
chmod 400 test-instance-key-pair.pem
```

> 프라이빗 키 소유자만 읽기 권한 필요

<br>

#### 네트워크 설정

- 기본 VPC/서브넷 선택
- 퍼블릭 IP 자동 할당 설정 확인
- 테스트 목적이라면 기본값 유지

<br>

#### 보안 그룹 설정

- 기본 생성된 보안 그룹에 SSH 허용 규칙 추가
- 모든 위치 허용은 테스트에서만 사용 권장

<br>

#### 스토리지 구성

- 루트 EBS 볼륨 기본값 유지
- 프리티어 기준 초과하지 않도록 설정 유지

<br>

#### 인스턴스 시작

- 설정 검토 후 인스턴스 시작 선택
- 성공 배너 표시 ➡️ 인스턴스 시작 완료

---

## 인스턴스 찾기

1. 콘솔 좌측 메뉴에서 인스턴스 선택
2. 이름(Name) 열에서 앞서 지정한 이름으로 필터링
3. Instance ID 또는 상태 확인

---

## 인스턴스 구성 보기

1. 인스턴스를 체크하고 작업 ➡️ 세부 정보 보기
2. 주요 필드 확인
   - AMI ID 및 이름
   - 인스턴스 유형
   - 키 페어 이름
   - VPC/서브넷 ID
   - 퍼블릭 IPv4 주소
   - 보안 그룹 정보
   - 스토리지 볼륨 정보
   - 상태(Running)
  
---

## 인스턴스 연결(SSH)

- Linux 기준 SSH 접속

```bash
ssh -i test-instance-key-pair.pem ec2-user@<퍼블릭-IP>
```

- 퍼블릭 IP는 EC2 상세 정보에서 확인
- SSH 보안그룹 규칙이 필요

  - Terminate 시 운영 중 데이터는 삭제됨

## 전체 비유 프레임

> 서비스를 위한 건물을 짓는다고 가정

#### AMI = 설계도 + 내부 인테리어 패키지

> AMI는 빈 땅에 지을 건물의 기본 형태가 이미 완성된 모델하우스

##### 실생활 비유

- 아파트 기본형
- 사무실 기본 구조
- 이미 전기/수도 배선이 완료된 상태

<br>

##### 서버 관점

- OS가 깔려 있음
- 기본 라이브러리 설치됨
- 보안 패치 적용됨

> 즉, 어떤 상태에서 시작할 것인가?

---

#### 인스턴스 유형 = 건물 크기와 구조

> 이건 건물의 평수와 하중 설계

##### 실생활 비유

- 10평 원룸
- 50평 사무실
- 200평 공장

<br>

##### 서버 대응

- CPU 코어 수
- 메모리 크기
- 네트워크 대역폭

> 즉, 이 건물에 얼마나 많은 사람이 들어올 수 있는가?

---

#### 네트워크 / 서브넷 / 퍼블릭 IP = 건물 위치와 도로 연결

##### 실생활 비유

- 도심 한복판(Public Subnet)
- 외곽 산업단지(Private Subnet)
- 골목길 안쪽(외부 접근 제한)

<br>

##### 퍼블릭 IP

- 건물에 붙어 있는 간판 주고
- 누구나 길찾기에서 검색 가능

<br>

##### Private Subnet

- 내부 직원만 출입 가능한 단지

> 즉, 이 건물이 어디에 있고 누가 찾아올 수 있는가?
