## 핵심 요약

- AWS EC2에서 테스트용 EC2 인스턴스 생성 ➡️ 찾기 ➡️ 구성 보기 ➡️ 접속 ➡️ 중지까지의 전체 흐름을 단계별로 설명
- 프리티어 가능 옵션을 기본으로 사용 추천
- 주요 설정: AMI 선택, 인스턴스 유형, 키페어, 네트워크/보안그룹, 스토리지

---

## 자습서 개요

- 목적: 테스트 용도의 EC2 인스턴스를 빠르게 시작하고 연결하는 방법 습득
- Linux 인스턴스를 기준으로 설명
- 프리티어 범위 내에서 비용 발생 없이 실습 가능 조건 설명 포함

---

## 사전 준비

- AWS 계정 로그인 완료
- IAM 사용자 권한 충분(EC2 인스턴스 생성/접속 등)
- SSH 접속을 위한 키 페어 생성 준비(다운로드 포함)

---

## 테스트 인스턴스 시작 절차

#### AWS EC2 콘솔 열기

1. AWS Management Console 로그인
2. 서비스 검색에서 EC2 선택
3. EC2 대시보드 진입

<br>

#### EC2 인스턴스 시작 마법사 실행

1. 인스턴스 시작 선택
2. 인스턴스 시작 웹 기반 양식 열림

<br>

#### 인스턴스 이름 및 태깅

- Name 태그 입력
- 태그는 인스턴스 목록에서 필터링/식별에 도움

<br>

#### AMI 선택

- 기본적으로 Amazon Linux 2023 AMI가 선택됨
- 프리티어 사용 가능 표시가 있는 AMI 선택 권장

<br>

#### 인스턴스 유형 선택

- 기본 인스턴스 유형 유지
- 프리 티어 범위 내 선택으로 비용 관리

<br>

#### 키 페어 설정

- 기존 키 페어 선택 또는 새 키 페어 생성 선택
- 프라이빗 키 파일 다운로드 후 로컬 저장 필수
- 추후 SSH 접속 시 사용

```bash
chmod 400 test-instance-key-pair.pem
```

> 프라이빗 키 소유자만 읽기 권한 필요

---

#### 네트워크 설정

- 기본 VPC/서브넷 선택
- 퍼블릭 IP 자동 할당 설정 확인
- 테스트 목적이라면 기본값 유지

--- 

#### 보안 그룹 설정

- 기본 생성된 보안 그룹에 SSH 허용 규칙 추가
- 모든 위치 허용은 테스트에서만 사용 권장

---

#### 스토리지 구성

- 루트 EBS 볼륨 기본값 유지
- 프리티어 기준 초과하지 않도록 설정 유지
