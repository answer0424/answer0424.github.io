## 핵심 요약

- EC2 인스턴스 시작 시 설정해야 하는 주요 파라미터 항목 정리
- AWS 콘솔의 인스턴스 시작 마법사 또는 시작 템플릿에서 사용할 수 있는 모든 구성값을 설명
- 대부분 필드는 기본값 제공 ➡️ 필요 시 커스터마이즈 가능
- 키 페어는 기본값이 없으며 반드시 지정 또는 생성 해야 함

---

## 파라미터 그룹 & 설명

#### 이름 및 태그

- 인스턴스 이름 지정은 태그로 처리됨
  - Key: `Name`, Value: 지정 이름
- 인스턴스, 연결된 볼륨, 네트워크 인터페이스 등 리소스별로 태그 추가 가능
- 태그는 식별, 필터링, 비용 할당 태그 등으로 활용됨

<br>

#### 애플리케이션 및 OS 이미지(AMI)

- Amazon Machine Image(AMI)는 인스턴스의 OS + 기본 파일 시스템 이미지를 의미함
- AMI는 리전별로 다르며 Linux/Windows 등 다양한 옵션 존재
- 선택된 AMI에 따라 OS, 소프트웨어 구성, 아키텍처 등이 결정됨

<br>

#### 인스턴스 유형

- CPU/vCPU, 메모리, 스토리지, 네트워크 성능 등을 결정하는 자원 크기
- 예: `t3.micro`, `m5.large`, `c7g.xlarge`등

<br>

#### 키 페어(Key Pair)

- SSH 인증 또는 RDP에 사용되는 키 페어
- 기본값이 제공되지 않으며 생성 또는 선택 필요
  - 키 다운로드 후 보안 저장 필수
- 콘솔, CLI, SDK 모두 지원

<br>

#### 네트워크 설정

- VPC, 서브넷 선택
- 퍼블릭 IP 자동 할당 여부
- 네트워크 인터페이스 구성
- 보안 그룹 선택
  - SSH, RDP, HTTP 등 포트 규칙 지정
- 세부 네트워크 ACL, 프라이버시 요구 사항 고려

<br>

#### 스토리지 구성

- 루트 디바이스 크기, 유형 지정
  - gp3, gp2, io2 등
- 추가 EBS 볼륨 연결
- 스냅샷 기반 볼륨 생성
- 암호화 설정
- IOPS/Throughput 옵션
- 필요 시 Provisioned IOPS 설정
