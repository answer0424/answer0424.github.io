## 핵심 요약

1. 인스턴스가 running 상태
2. 네트워크 접근 가능
3. 보안 그룹에서 접속 포트 허용
4. 접속 방식에 맞는 인증 정보
5. 올바른 사용자 계정

---

## 1. 인스턴스 상태 확인

```text
EC2 Console ➡️ Instances ➡️ Instance State
```

| 상태 | 의미 |
| -|-|
| running | 정상 실행 |
| stopped | 중지 |
| pending | 시작 중 |

---

## 2. 네트워크 접근 가능 여부

#### 인터넷 접속 방식

- Public IPv4 또는 IPv6 주소
- 인터넷 게이트웨어가 연결된 VPC
- Public Subnet

<br>

#### Private 네트워크 접속

- VPN
- AWS Direct Connect
- VPC Peering
- EC2 Instance Connect Endpoint

---

## 3. 보안 그룹 설정

| OS | 기본 접속 포트 |
|-|-|
| Linux | SSH |
| Windows | RDP |


```text
Type: SSH
Port: 22
Source: 내 IP 또는 허용 네트워크
```

---

## 4. 인증 정보 준비

#### Linux

```text
ssh -i my-key.pem ec2-user@<public-ip>
```

<br>
 
#### Windows

- Administrator 암호 + RDP 클라이언트

<br>

#### EC2 Instance Connect

- IAM 권한 필요
- SSH 키를 인스턴스 메타데이터로 전달하여 인증
