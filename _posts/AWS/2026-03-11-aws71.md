## 핵심 요약

- Linux EC2 인스턴스는 SSH 프로토콜을 사용해 접속
- 접속을 위해 필요한 핵심요소
  - EC2 Key Pair(.pem)
  - 퍼블릭 IP 또는 퍼블릭 DNS
  - SSH 클라이언트
  - AMI에 맞는 사용자 계정
- 기본 SSH 접속 명령 예:

```bash
ssh -i my-key.pem ec2-user@<EC2-Public-IP>
```

---

## 1. SSH 연결 사전 조건

> SSH로 EC2 Linux 인스턴스에 연결하기 전에 다음 조건이 충족되어야 함

#### 인스턴스 상태

- 인스턴스가 running 상태
- 상태 검사 통과 필요

<br>

#### 보안 그룹 설정

> EC2 보안그룹에서 SSH 인바운드 규칙이 허용되어야 함

| 항목 | 값 |
|-|-|
| Type | SSH |
| Port | 22 |
| Source | 내 IP 또는 허용 네트워크 |

<br>

#### 프라이빗 키 준비

> EC2 인스턴스 생성 시 다운로드 한 Key Pair(.pem) 필요

```bash
chmod 400 my-key.pem
```

<br>


#### SSH 클라이언트 설치

> 대부분 운영체제에 기본 포함되어 있음

```bash
ssh
```

> 명령이 인식되지 않으면 SSH 클라이언트를 설치해야 함

---

## 2. SSH로 Linux 인스턴스 접속

#### 기본 명령

```bash
ssh -i my-key-pair.pem ec2-user@ec2-public-dns
```

- 구성 요소 설명

| 옵션 | 설명 |
|-|-|
| -i | Key Pair 파일 |
| ec2-user | 로그인 사용자 |
| ec2-public-dns | 인스턴스 주소 |

---

## 3. AMI 별 기본 사용자 계정

> AMI에 따라 로그인 사용자 이름이 다름

| AMI | 기본 사용자 |
|-|-|
| Amazon Linux | ec2-user |
| Ubuntu | ubuntu |
| RHEL | ec2-user 또는 root |
| CentOS | centos |
| Debian | admin 또는 root |
