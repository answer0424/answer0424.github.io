## 핵심 요약

- Windows에서 EC2 Linux 인스턴스에 접속하여면 SSH 클라이언트가 필요
- AWS에서는 OpenSSH 또는 PuTTY 사용을 안내
- 접속에 필요한 핵심 요소
  - EC2 Key Pair(.pem)
  - 퍼블릭 IP 또는 퍼블릭 DNS
  - PuTTY SSH 클라이언트
  - AMI에 맞는 사용자 계정

---

## 1. Windows에서 사용할 수 있는 SSH 방법

| 방법 | 특징 |
|-|-|
| OpenSSH | Windows 기본 SSH |
| PuTTY | Windows 전용 SSH 클라이언트 |
| WSL + SSH | Linux 환경에서 SSH 사용 |

---

## 2. 방법 1 - Windows OpenSSH 사용 (권장)

#### 1) OpenSSH 설치

> PowerShell 관리자 권한 실행

```powershell
Add-WindowsCapability -Online -Name OpenSSH.Client~~~~0.0.1.0
```

<br>

#### 2) Key Pair 준비

> EC2 인스턴스 생성 시 다운로드 한 `.pem` 파일 사용

```powershell
C:\Users\user\Downloads\my-key.pem
```

<br>

#### 3) SSH 접속

```bash
ssh -i my-key.pem ec2-user@<public-ip>
```

- 예

```bash
ssh -i my-key.pem ec2-user@3.36.xxx.xxx
```

---

## 3. 방법 2 - PuTTY 사용

> PuTTY는 `.pem` 키를 직접 사용하지 못하므로
>
> PuTTYgen으로 `.ppk` 변환이 필요

#### 1) PuTTYgen으로 키 변환

1. PuTTYgen 실행
2. Load ➡️ `.pem` 파일 선택
3. Save private key ➡️ `.ppk` 파일 저장

<br>

#### 2) PuTTY 접속 설정

| 항목 | 값 |
|-|-|
| Host Name | ec2-user@<public-ip> |
|Port | 22 |


---

## 4. 방법 3 - WSL 사용

> Windows에서 Linux 환경을 사용하려면 WSL을 사용할 수 있음

```powershell
wsl --install
```

> WSL에서 SSH 접속

```bash
ssh -i my-key.pem ec2-user@<public-ip>
```

---

## 5. AMI별 기본 사용자

| AMI | Username |
|-|-|
| Amazon Linux | ec2-user |
| Ubuntu | ubuntu |
| RHEL | ec2-user 또는 root |
| CentOS | centos |
| Debian | admin 또는 root |

---

## 6. 연결 문제 체크리스트

| 문제 | 원인 |
|-|-|
| Connection timed out | Security Group 22 포트 차단 |
| Permission denied | 키 파일 문제 |
| Host unreachable | Public IP 없음 |
| Authentication failed | 잘못된 username |
