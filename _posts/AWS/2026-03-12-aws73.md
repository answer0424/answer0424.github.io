## 핵심 요약

- SCP는 SSH 기반 파일 전송 방식
- 로컬 컴퓨터 ↔️ EC2 Linux 인스턴스 간 파일 업로드/다운로드에 사용
- SSH 접속과 동일하게 Key Pair(.pem)를 사용하여 암호화된 전송이 이루어짐
- 단순 파일 복사 작업에 적합하며, 대용량/동기화 작업에는 rsync가 더 효율적일 수 있음

---

## 1. SCP 개념

- SSH 기반 파일 전송 프로토콜
- 로컬 시스템과 원격 서버 간 파일 복사 가능
- SSH 인증 방식 사용
- 대부분 OpenSSH 패키지에 기본 포함

| 항목 | 설명 |
|-|-|
| 프로토콜 | SSH |
| 인증 | Key Pair |
| 사용목적 | 단순 파일 복사 |
| 보안 | SSH 암호화 |

--- 


## 2. 사전조건

> SCP로 EC2 인스턴스에 파일을 전송하려면 다음 조건이 필요

#### 1. EC2 인스턴스 실행 상태

- 인스턴스 상태: running

<br>

#### 2. SSH 접근 가능

> 보안 그룹에 SSH 허용 필요

| 설정 | 값 |
|-|-|
| Protocol | TCP |
| Port | 22 |
| Source | 접속 IP |


<br>

#### 3. Key Pair

> EC2 생성 시 다운로드 한 키 파일 필요

```bash
chmod 400 my-key.pem
```

<br>

#### 4. SCP 클라이언트

> 대부분 OS에 기본 설치되어 있음

---

## 3. 파일 업로드(Local ➡️ EC2)

> 로컬 파일을 EC2 인스턴스로 전송

```bash
scp -i my-key.pem file.txt ec2-user@<public-ip>:/home/ec2-user/
```

- 예

```bash
scp -i my-key.pem test.txt ec2-user@3.35.xxx.xxx:/home/ec2-user/
```

- 구성

| 옵션 | 설명 |
|-|-|
| -i | Key Pair |
| file.txt | 업로드할 파일 |
| ec2-user | SSH 사용자 |
| public-ip | EC2 주소 |
| /home/ec2-user | 대상 경로 |

---

## 4. 파일 다운로드 

> EC2 인스턴스에서 로컬로 파일 복사

```bash
scp -i my-key.pem ec2-user@<public-ip>:/home/ec2-user/file.txt ./
```


- `.`은 현재 디렉토리를 의미

---

## 5. 디렉터리 전송

```bash
scp -i my-key.pem -r myfolder ec2-user@<public-ip>:/home/ec2-user/
```

- 옵션

| 옵션 | 설명 |
|-|-|
| -r | recursive (폴더 전송) |

---

## 6. AMI별 기본 사용자

| AMI 유형 | 기본 사용자 |
|-|-|
| Amazon Linux 2 | ec2-user |
| Ubuntu | ubuntu |
| RHEL | ec2-user 또는 root |
| CentOS | centos |
| Debian | admin 또는 root |

--- 

## 7. 대용량 파일 전송

- 대용량 데이터 전송
- 반복 동기화
- 네트워크 효율 필요

> 이유

- SCP는 단순 복사 방식
- rsync는 변경된 부분만 전송 가능
