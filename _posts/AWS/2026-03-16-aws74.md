## 핵심 요약

- EC2 Linux 인스턴스는 시작 시ㅣ 기본 Linux 시스템 사용자가 생성됨
- 필요에 따라 사용자를 추가하거나 삭제하여 접근 권한을 관리할 수 있음
- 기본적으로
  - SSH 키 기반 로그인 사용
  - 비밀번호 인증 및 root 로그인 비활성화
  - sudo 권한 활성화 상태임
- EC2의 Linux 사용자 계정은 AWS IAM 사용자와 별개의 개념임

---

## 1. 기본 사용자

> EC2 Linux 인스턴스는 AMI 종류에 따라 기본 로그인 사용자 이름이 다름

| AMI | 기본 사용자 |
|-|-|
| Amazon Linux | ec2-user |
| Ubuntu | ubuntu |
| Debian | admin 또는 root |
| CentOS | centos |
| Fedora | fedora |
| RHEL | ec2-user 또는 root |
| SUSE | ec2-user 또는 root |
| Oracle Linux | ec2-user |
| Bitnami AMI | bitnami |

---

## 2. 사용자 관리 고려 사항

- 이유
  - 사용자별 작업 공간 분리
  - 권한 관리 용이
  - 보안 강화

> 특히 여러 사람이 동일한 기본 계정을 고유하면 시스템 손상 위험이 증가함

---

## 3. EC2 Linux 사용자 생성

#### 1) 사용자 생성

```bash
sudo adduser newuser
```

```bash
sudo useradd newuser
```

<br>

#### 2) sudo 권한 부여

```bash
sudo usermod -aG wheel newuser
```

> Amazon Linux / RHEL 기반에서는 wheel 그룹이 관리자 권한을 가짐

<br>

#### 3) SSH 접속용 디렉토리 생성


```bash
sudo mkdir /home/newuser/.ssh
```

- 권한 설정

```bash
sudo chmod 700 /home/newuser/.ssh
```

<br>

#### 4) SSH 공개키 등록


```bash
sudo nano /home/newuser/.ssh/authorized_keys
```

- 사용자의 SSH public key를 추가함

```bash
sudo chmod 600 /home/newuser/.ssh/authorized_keys
```

<br>

#### 5) 소유권 설정

```bash
sudo chown -R newuser:newuser /home/newuser/.ssh
```

---

## 4. 사용자 삭제

```bash
sudo userdel newuser
```

- 홈 디렉토리 삭제

```bash
sudo userdel -r newuser
```

<br>

## 5. SSH 접근 관리

> Linux 사용자에게 SSH 접근을 제공하려면 다음 중 하나를 사용함

#### 방법

| 방법 | 설명 |
|-|-|
| SSH Key 공유 | authorized_keys 사용 |
| EC2 Instance Connect | SSH 키 자동 관리 |

> EC2 Instance Connect를 사용하면 SSH 키를 직접 관리하지 않고도 접근 권한을 제공할 수 있음

<br>

## 6. 사용자 관리 흐름

1. EC2 인스턴스 접속
2. Linux 사용자 생성
3. SSH 키 등록
4. SSH 접속 허용

---

## 7. IAM 사용자 vs Linux 사용자

| 구분 | IAM 사용자 | Linux 사용자 |
|-|-|-|
| 관리 위치 | AWS 계정 | EC2 OS 내부 |
| 목적 | AWS 리소스 접근 | 서버 로그인 |
| 인증 방식 | Access Key, IAM | SSH Key |

--- 

## 정리

1. 인스턴스에는 AMI 기반 기본 사용자가 존재
2. 필요 시 Linux 시스템 사용자 생성 가능
3. SSH 접근을 위해 authorized_keys에 공개키 등록
4. 여러 사용자 환경에서는 개별 계정 생성이 보안상 권장
5. IAM 사용자와 Linux 사용자 계정은 별개
