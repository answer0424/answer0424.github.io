## 핵심 요약

- EC2 Linux 인스턴스는 시작 시ㅣ 기본 Linux 시스템 사용자가 생성됨
- 필요에 따라 사용자를 추가하거나 삭제하여 접근 권한을 관리할 수 있음
- 기본적으로
  - SSH 키 기반 로그인 사용
  - 비밀번호 인증 및 root 로그인 비활성화
  - sudo 권한 활성화 상태임
- EC2의 Linux 사용자 계정은 AWS IAM 사용자와 별개의 개념임

---

## 1. 기본 사용자

> EC2 Linux 인스턴스는 AMI 종류에 따라 기본 로그인 사용자 이름이 다름

| AMI | 기본 사용자 |
|-|-|
| Amazon Linux | ec2-user |
| Ubuntu | ubuntu |
| Debian | admin 또는 root |
| CentOS | centos |
| Fedora | fedora |
| RHEL | ec2-user 또는 root |
| SUSE | ec2-user 또는 root |
| Oracle Linux | ec2-user |
| Bitnami AMI | bitnami |

---

## 2. 사용자 관리 고려 사항

- 이유
  - 사용자별 작업 공간 분리
  - 권한 관리 용이
  - 보안 강화

> 특히 여러 사람이 동일한 기본 계정을 고유하면 시스템 손상 위험이 증가함

---

## 3. EC2 Linux 사용자 생성

#### 1) 사용자 생성

```bash
sudo adduser newuser
```

```bash
sudo useradd newuser
```

<br>

#### 2) sudo 권한 부여

```bash
sudo usermod -aG wheel newuser
```

> Amazon Linux / RHEL 기반에서는 wheel 그룹이 관리자 권한을 가짐
