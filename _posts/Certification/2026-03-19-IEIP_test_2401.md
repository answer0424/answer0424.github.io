## 2번

- 1단계 디렉터리 구조: 가장 간단하고 모든 파일이 하나의 디렉터리 내에 위치
- 2단계 디렉터리 구조: 중앙에 마스터 파일 디렉터리가 있고, 그 아래에 사용자별로 서로 다른 파일 디렉터리가 있는 구조
- 트리 디렉터리 구조: 하나의 루트 디렉터리와 여러 개의 종속(서브) 디렉터리로 구성된 구조
- 비순환 그래프 디렉터리 구조: 하위 파일이나 하위 디렉터리를 공동으로 사용할 수 있는 것으로 사이클이 허용되지 않는 구조

## 16번

- 피어-투-피어 패턴: 노드와 간선으로 구성됨
- 레이어 패턴: 계층 모델이라고도 함
- 파이프-필터 패턴: 서브 시스템이 입력 데이터를 받아 처리하고 결과를 다음 서브 시스템으로 넘겨주는 과정을 반복함
- 모델-뷰-컨트롤러 패턴: 3개의 서브 시스템으로 구성되어 있음

<br>

## 17번

- SDLC(Software Development Life Cycle): 소프트웨어 개발 방법론의 바탕이 되는 것으로, 소프트웨어를 개발하기 위해 정의하고 운용, 유지보수 등의 과정을 각 단계별로 나눈 것 
- Secure Coding: 소프트웨어 구현 단계에서 발생할 수 있는 보안 취약점들을 최소화하기 위해 보안 요소들을 고려하며 코딩하는 것을 의미하며, 보안 취약점을 사전에 대응하여 안정성과 신뢰성을 확보하기 위해 사용
- CLASP(Comprehensive, Lightweight Application Security Process): Secure Software사에서 개발하였으며, SDLC의 초기 단계에서 보안을 강화하기 위해 개발된 방법론
- OWASP(Open Web Application Security Project): 웹 정보 노출이나 악성 코드, 스크립트, 보안이 취약한 부분을 연구하는 비영리 단체
