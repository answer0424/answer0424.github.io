## 상호 배제를 올바로 구현하기 위한 요구 조선에 대한 설명으로 틀린 것은?

1. 두개 이상의 프로세스들이 공유 데이터에 접근하여 동시에 수행할 수 있어야 한다
2. 임계구역 바깥에 있는 프로세스가 다른 프로세스의 임계 구역 진입을 막아서는 안된다
3. 어떤 프로세스도 임계 구역으로 들어가는 것이 무한정 연기되어서는 안된다
4. 임계 구역은 특정 프로세스가 독점할 수 없다

> 3번 / 상호배제는 한 번에 한 개의 프로세스만이 공유 자원을 사용할 수 있도록 하는 것이므로 두 개 이상의 프로세스들이 공유 데이터를 동시에 접근할 수 없음

## OSI 7 계층 중 IP 프로토콜과 관련된 계층으로 네트워크 연결 관리 및 데이터의 교환 및 중계 기능을 수행하는 계층은?

2. 데이터 링크 계층
3. 네트워크 계층

> 3번 
>
> 물리 계층: 전송에 필요한 두 장치 간의 실제 접속과 절단 등 기계쩍, 전기적, 기능적, 절차적 특성을 정의함

> 데이터 링크 계층: 두 개의 인접한 개발 시스템들 간에 신뢰성 있고 효율적인 정보 전송을 할 수 있도록 함

## 하나의 프로세스가 자주 참조하는 페이지의 집합을 의미하며 이런 페이지 집합이 적재되면 프로세스는 한동안 페이지 포트없이 실행될 수 있다. 이런 페이지 집합을 무엇이라고 하는가?

1. Working Set
2. Critical Section
3. Paging
4. Fragmentation

> 1번
>
> Critical Section: 다중 프로그래밍 운영체제에서 여러 개의 프로세스가 공유하는 데이터 및 자원에 대하여 어느 한 시점에서는 하나의 프로세스만 지원 또는 데이터를 사용하도록 지정된 공유 자원을 의미함
>
> Paging 기법: 가상 기억장치에 보관되어 있는 프로그램과 주기억장치의 영역을 동일한 크기로 나눈 후 나눠진 프로그램을 동일하게 나눠진 주기억장치의 영역에 적재시켜 실행하는 기법
>
> Fragmentation: 분할된 주기억장치에 프로그램에 할당하고 반납하는 과정을 반복하면서 사용되지 않고 남는 기억장치의 빈공간 조각을 의미함

<br>

## 다음 객체 지향 기법에 대한 설명에 해당하는 것은?

```text
메시지에 의해 객체가 연산을 수행하게 될 때 하나의 메시지에 대해 각 객체가 가지고 있는 고유한 방법으로 응답할 수 있는 능력
```

1. Encapsulation
2. Abstraction
3. Inheritance
4. Ploymorphism

> 4번 
>
> Encapsulation: 데이터와 데이터를 처리하는 함수를 하나로 묶는 것을 의미
>
> Abstraction: 문제의 전체적이고 포괄적인 개념을 설계한 후 차례로 세분화하여 구체화시켜 나가는 것
>
> Inheritance: 이미 정의된 상위 클래스의 모든 속성과 연산을 하위 클래스가 물려받는 것

## 개체 집합에 대한 속성 관계를 표시하기 위해 개체를 노드로 표현하고 개체 집합들 사이의 관계를 링크로 연결한 트리 형태의 자료 구조 모델은?

1. 망 데이터 모델
2. 계층 데이터 모델
3. 관계 데이터 모델
4. 객체지향 데이터 모델

>
> 망형 데이터 모델(Network Data Model): 그래프를 이용해서 데이터 논리 구조를 표현한 모델로, CODASYL이 제안하여 CODASYL DBTG 모델이라고도 함
>

>
