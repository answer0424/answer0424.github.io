## 시스템 아키텍처에 대한 설명으로 옳은 것은?

1. 유비쿼터스 시스템에는 계층형 아키텍처가 적합하다
2. 분산 시스템은 적합한 아키텍처는 MVC
3. 음성 인식, 신호 해석 등에 적합한 아키텍처는 파이트-필터
4. 데이터 중심의 소프트웨어에는 저장소 유형의 아키텍처 모델이 제시되어야 함

> 4번
>
> 유비쿼터스 시스템에는 주로 사용되는 MVC 아키텍처가 적합
>
> 분산 시스템에 적합한 아키텍처는 브로커 패턴임
>
> 음성 인식, 신호 해석등에 적합한 아키텍처는 블랙보드 패턴임

## UI 시스템에 대한 설명으로 옳지 않은 것은?

1. 입력 데이터를 사전에 검증하는 역할을 수행
2. 잘못된 입력으로 발생하는 예외를 처리
3. 오류에 대한 구체적인 시스템 정보를 메시지로 제공함
4. 사용자에게 입력창과 프롬프트, 도움말 등을 제공

> 3번 / 오류에 대한 구체적인 시스템 정보가 노출되면 보안에 악영향을 끼침

<br>

## 인터페이스 클래스에 대한 설명으로 옳지 않은 것은?

1. 일반적인 메소드와 변수를 선언할 수 있다
2. 객체 변수를 생성할 수 없다
3. 다중 상속이 가능
4. 상속 시 implements를 사용

> 1번 / 인터페이스 클래스에서는 일반적인 메소드나 변수 대신, 추상 메소드와 상수와 같이 값을 변경할 수 없는 변수만을 선언하여 사용

<br>

## 소프트웨어 개발 프레임워크에 대한 설명으로 옳지 않은 것은?

1. 모듈화로 인해 설계 및 구현의 변경에 따른 영향을 극소화시킴
2. 재사용된 모듈을 사용하여 품질을 향상시키고, 용량을 감소시킴
3. 객체들의 통제를 프레임워크에 넘김으로써 생산성을 향상시킴
4. 인터페이스의 확장이 자유로워 다양한 기능을 가진 소프트웨어 개발이 가능

> 2번 / 재사용된 모듈은 예산 절감, 생산성 향상, 품질 보증 등의 이점이 있지만, 용량 감소에 대한 이점은 없음

<br>

## 매크로 함수와 일반 함수에 대한 설명으로 옳지 않은 것은?

1. 매크로 함수는 컴파일 전에 치환되므로 코드의 용량이 증가함
2. 일반 함수와 매크로 함수는 모두 인수의 자료형을 입력해야 함
3. 일반 함수는 코드의 용량을 증가시키지 않음
4. 매크로 함수는 일반 함수에 비해 속도가 빠름

> 2번 / 매크로 함수는 인수의 자료형을 입력하지 않음

<br>

## 관계대수의 일반 집합 연산자에 대한 설명으로 옳지 않은 것은?

1. 교집합, 합집합은 두 릴레이션 합병이 가능해야 함
2. 차집합, 카티션 프로덕트 등이 있음
3. 카티션 프로덕트의 연산자는 π로 표현함
4. 연산을 위해 피연산자 두개가 필요

> 3번 / 카티션 프로덕트의 연산자는 x로 표현
