## 운영체제 운용 기법

1. 일괄 처리 시스템: 데이터를 모아서 한번에 처리
2. 다중 프로그래밍 시스템: 하나의 CPU와 주기억장치를 이용해 여러 개의 프로그램을 동시 처리
3. 시분할 시스템: 라운드 로빈이라고 하며, 모든 작업이 동시에 진행되는 것처럼 보임
4. 다중 처리 시스템: 여러 개의 CPU와 주기억장치를 이용해 여러 개의 프로그램을 동시 처리
5. 실시간 처리 시스템: 데이터 발생 즉시 처리
6. 다중 모드 처리: 일괄 처리, 시분할, 다중 처리, 실시간 처리를 한번에 모아둔 시스템
7. 분산 처리 시스템: 여러 개의 컴퓨터를 통신 회선으로 연결하여 하나의 작업을 처리

## 데이터베이스 설계

1. 개념적 설계: 현실 세계에 대한 인식을 추상적 개념을 표현
2. 논리적 설계: 특정 DBMS가 지원하는 논리적 자료 구조로 변환시키는 과정
3. 물리적 설계: 논리적 구조로 표현된 데이터를 물리적 구조의 데이터로 변환시키는 과정

## 관계형 데이터베이스의 구조

- 튜플: 릴레이션을 구성하는 각각의 행(**카디널리티, 기수, 대응수**)
- 속성: 데이터베이스를 구성하는 가장 작은 논리적 단위(**디그리, 차수**)
- 도메인: 하나의 애트리뷰트가 취할 수 있는 같은 타입의 원자값들의 집합

## 트랜잭션 분석

1. Atomicity(원자성): DB에 모두 반영되든지 모두 반영되지 않도록 해야 함
2. Consistency(일관성): 트랜잭션 실행이 성공하면 언제나 일관성있는 상태를 유지해야 함
3. Isolation(독립성): 하나의 트랜잭션이 실행 중에 다른 트랜잭션이 끼어들 수 없음
4. Durability(지속성): 시스템이 고장나더라도 영구적으로 반영되어야 함

## OSI 참조모델

1. 물리 계층: 전송에 필요한 두 장치 간의 실제 접속과 절단
   1. 관련 장비: 리피터, 허브
2. 데이터 링크 계층: 두 개의 인접한 개방 시스템들 간에 신뢰성 있고 효율적인 정보 전송을 할 수 있도록 시스템 간 연결 설정과 유지 및 종료를 담당, 오류 제어 기능, 순서 제어 기능
   1. 표준: HDLC, LAPB, LLC, MAC, LAPD, PPP
   2. 관련 장비: 랜카드, 브리지, 스위치
3. 네트워크 계층: 개방 시스템들 간의 네트워크 연결을 관리하는 기능과 데이터 교환 및 중계 기능
   1. 표준: X.25, IP
   2. 관련 장비: 라우터
4. 전송 계층: 균일한 데이터 전송 서비스를 제공함으로써 종단 시스템 간의 투명한 데이터 전송을 가능하게 함
   1. 표준: TCP, UDP
   2. 관련 장비: 게이트웨이
5. 세션 계층: 송수신 측 간의 관련성을 유지하고 대화 제어를 담당
6. 표현 계층: 코드 변환, 데이터 암호화, 데이터 압축, 구문 검색, 정보 형식 변환, 문맥관리 기능
7. 응용 계층: 사용자가 OSI 환경에 접근할 수 있도록 서비스를 제공

## 네트워크 관련 장비

- 허브: 회선을 통합하여 관리하고 신호 증폭 기능을 하는 리피터의 역할을 포함
- 리피터: 수신한 신호를 재생시키거나 출력 전압을 높여 전송하는 장치
- 브리지: LAN과 LAN을 연결하거나 LAN 안에서의 컴퓨터 그룹을 연결하는 장치이며 트래픽 병목 현상을 줄일 수 있음
- 스위치: LAN과 LAN을 연결하여 훨씬 더 큰 LAN을 만드는 장치
- 라우터: 브리지와 같은 기능을 하며, 데이터 전송의 최적 경로를 선택하는 기능이 추가된 장치
- 게이트웨이: 프로토콜 구조가 다른 네트워크를 연결하는 장치

## TCP/IP

#### 응용 계층 주요 프로토콜

- FTP: 파일을 주고받을 수 있도로 하는 원격 파일 전송 프로토콜(21)
- SMTP: 전주 우편을 교환하는 서비스(25)
- TELNET: 멀리 떨어져 있는 컴퓨터에 접속하여 원격으로 사용할 수 있도록 하는 서비스(23)
- SNMP: TCP/IP의 네트워크 관리 프로토콜(161, 162)
- DNS: 도메인 네임을 IP 주소로 변환시켜주는 시스템(53)
- HTTP: 월드 와이웹에서 HTML 문서를 송수신 하기 위한 표준 프로토콜(80)
- SSH: 원격 서버에 안전하게 접속할 수 있는 프로토콜(22)
- POP3: 메일 서버에 저장되어 있는 이메일을 내 컴퓨터로 가져오는 프로토콜(110)
- NNTP: 인터넷 뉴스를 읽고 게시하기 위한 프로토콜(119)
- IMAP: 메일 서버에 저장된 이메일을 서버와 동기화하여 관리하는 프로토콜(143)
- HTTPS: 웹 브라우저와 웹 서버가 암호화된 방식으로 데이터를 주고 받기 위한 프로토콜(443)

<br>

#### 전송 계층의 주요 프로토콜

- TCP: 양방향 신뢰성 있는 연결 서비스
- UDP: 비연결형 서비스
- RTCP: 패킷의 전송 품질을 제어하기 위한 제어 프로토콜

> TCP의 신뢰성 있는 연결 수립 및 해제
>
> 수립: 3-way-handshake
>
>   송신자 SYN -> 수신자 SYN ACK -> 송신자 ACK
>
> 해제: 4-way-handshake
>
>   송신자 FIN -> 수신자 ACK -> 수신자 FIN -> 송신자 ACK


<br>

#### 인터넷 계층의 주요 프로토콜

- IP: 연결형 데이터 그램 방식으로 사용하여 전송할 데이터에 주소를 지정하고 경로를 설정하는 역할
- ICMP: IP와 조합하여 통신 중에 발생하는 오류의 처리와 제어 메시지를 관리
- IGMP: 인터넷 그룹 관리 프로토콜
- ARP: IP 주소를 MAC 주소로 변환
- RARP: MAC 주소를 IP 주소로 변환

<br>

#### 네트워크 액세스 계층의 주요 프로토콜

- Ethernet
- IEEE 802
- HDLC: 비트 위주의 데이터 링크 제어 프로토콜
- X.25

## 경로 제어 / 트래픽 제어

#### 경로 제어

- IGP: 내부 게이트웨이 프로토콜
  - RIP: 가장 널리 사용되는 라우팅 프로토콜로 거리 벡터 라우팅 프로토콜이라고도 불림
  - OSPF: RIP의 단점을 보완한 프로토콜
- EGP: 자율 시스템 간의 라우팅
- BGP: EGP의 단점을 보완하기 위해 만들어짐
