## 사용성 테스트
- 사용성 테스트는 시스템에 구현된 UI를 **일반 사용자가 사용하면서 문제점이나 개선점을 도출해내는 방식의 테스트**
- UI의 만족도나 완성도, 효율성 등을 확인하고 향상시키기 위해 수행
- 사용자가 접근하고 이용하는 측면에서 선호할 만한 기능을 테스트
- 요구사항의 반영 여부를 점검
- 테스트 과정에서 발생한 새로운 요구사항은 소프트웨어 구현 과정에서 반영

> **사용자 인터페이스(UI)**는 **UI 설계 -> 사용성 테스트 -> UI 테스트**과정을 거쳐 구현
