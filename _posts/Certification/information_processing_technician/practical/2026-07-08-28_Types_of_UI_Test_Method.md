## UI 테스트

- UI 테스트는 **구현된 UI의 사용성을 검증하기 위해 테스트를 수행하고, 결과에 따라 개선 및 결과 보고서를 작성하는 행위 또는 그 절차를 의미**
- UI 테스트는 사용자가 미리 작성된 시나리오에 따라 직접 제품을 사용하면서 진행하는 사용자 중심의 테스트
- UI 테스트는 사용자의 요구사항과 행동을 관찰할 수 있는 유요한 진단 방법
- UI 테스트 중 발생되는 산출물은 사용자 메뉴얼 작성 시 중요한 참고 자료가 됨

> 사용자 인터페이스는 UI 설계 -> 사용성 테스트 -> UI 테스트

---

## UI 테스트 기법의 종류

#### 휴리스틱 평가

- 최소 3명 이상의 디자인 전문가가 사전에 작성한 원칙에 따라 제품을 평가하는 기법
- UI의 구현 정도에 관계없이 평가가 가능
- 전문가의 능력에 따라 평가 시간이나 수준이 달라짐
- 주로 Jakob Nielsen의 10개 원칙이 평가에 사용

<br>

#### 페이퍼 프로토타입

- 종이로 해당 서비스를 간단하게 만들어 실제 구현되는 것처럼 표현하고 이를 이용하여 테스트하는 평가방법
- 프로토타입 작성 시 포함되어야 할 중요한 내용을 체크리스트로 작성
- 테스트 기법 중 가장 빠른 방법으로 제품의 전반적인 컨셉과 흐름을 보여줌
- 테스트 참가자들이 최종 제품에 대한 기대를 갖지 않고 더 자유롭게 의견을 내는 경향이 있음
